# Data Model: Console Todo App

**Branch**: `001-console-todo-app` | **Date**: 2025-12-07 | **Spec**: specs/001-console-todo-app/spec.md
**Input**: Feature specification from `/specs/001-console-todo-app/spec.md`

## Summary

This document defines the data model for the "Console Todo App", primarily focusing on the `Task` entity as described in the feature specification and constrained by the project constitution.

## Entities

### Task

Represents a single todo item within the application.

**Attributes**:

*   **`id`**:
    *   **Type**: `int`
    *   **Description**: A unique identifier for the task. Automatically generated by the system.
    *   **Validation**: Must be a positive integer.
*   **`title`**:
    *   **Type**: `str`
    *   **Description**: A brief, descriptive title for the task.
    *   **Validation**: Cannot be empty. Maximum length (e.g., 255 characters) for practical purposes, though not explicitly specified in the spec.
*   **`description`**:
    *   **Type**: `str`
    *   **Description**: An optional, more detailed description of the task. Can be empty.
    *   **Validation**: None beyond standard string handling.
*   **`completed`**:
    *   **Type**: `bool`
    *   **Description**: Indicates whether the task has been completed (`True`) or is still pending (`False`).
    *   **Default**: `False`
*   **`created_at`**:
    *   **Type**: `str` (ISO 8601 formatted datetime string, e.g., "YYYY-MM-DDTHH:MM:SS.ffffff")
    *   **Description**: The timestamp when the task was initially created.
    *   **Validation**: Must be a valid ISO 8601 datetime string.
*   **`updated_at`**:
    *   **Type**: `str` (ISO 8601 formatted datetime string)
    *   **Description**: The timestamp when the task was last modified (e.g., title, description, or completed status changed).
    *   **Validation**: Must be a valid ISO 8601 datetime string. Automatically updated by the system on modification.

**Relationships**:

*   The `Task` entity is self-contained and does not have explicit relationships with other entities within this Phase I scope.

**State Transitions**:

*   A `Task` can transition from `completed=False` to `completed=True` (marked as complete).
*   A `Task` can transition from `completed=True` to `completed=False` (marked as incomplete).
*   Changes to `title` or `description` do not affect `completed` status directly but trigger an `updated_at` timestamp update.
